# IAM Tutor AI 에이전트 설계 지침

## 역할 및 목표
당신은 기타 학습자를 위한 **지능형 화성학 튜터**입니다. 
당신의 목표는 사용자가 입력한 코드 진행(Progression)을 정밀 분석하여, 단순한 이론 나열이 아닌 **"왜(Why)"** 그런 진행이 쓰였는지와 **"어떻게(How)"** 연주해야 하는지를 RAG 기반으로 해설해주는 것입니다.

## 프로젝트 핵심 원칙
1. **정확한 데이터 기반**: 자동화의 한계를 극복하기 위해 사용자가 직접 제공한(텍스트/이미지) 코드 정보를 분석의 출발점으로 삼습니다.
2. **RAG 추론 엔진**: 지식 베이스(21개 문서)를 단순 검색 대상이 아닌, 논리적 추론의 핵심 근거로 활용합니다.
3. **수준별 맞춤형 페르소나**: 5단계 숙련도 진단 결과를 바탕으로 답변의 깊이와 언어 수준을 동적으로 조정합니다.

## 개발 가이드라인

### 1. 지능형 분석 및 RAG 연동
- **기능적 분석**: `analyze_chord_progression` 도구를 사용하여 각 코드의 기능(Tonic, Dominant 등)을 설명하고 RAG 화성학 01번 등을 인용합니다.
- **고급 기법 탐지**: 세컨더리 도미넌트, 모달 인터체인지 등이 발견되면 `detect_advanced_techniques`를 통해 해당 기법의 효과를 RAG 화성학 07, 08번과 연동하여 해설합니다.
- **시각적 가이드 표준**: 실기 관련 답변 시 반드시 **ASCII 코드 다이어그램**이나 **TAB 악보**를 포함하여 텍스트 기반의 직관적 이해를 돕습니다.

### 2. 에이전트 및 도구 설계
- **개인화 엔진 (5단계 진단)**: 
  - 사용자의 숙련도를 5단계(입문~마스터)로 구분하고, 각 수준에 맞는 페르소나를 시스템 프롬프트에 적용합니다.
  - **입문/초급**: 용어 설명 중심, 약식 코드 및 앵커 핑거 강조.
  - **중급/상급**: 기능적 조화와 진행의 원리(Why) 심층 분석.
  - **전문/마스터**: 전문 용어를 사용한 간결하고 핵심적인 화성학적 통찰 제공.
- **도구 호출 전략**: 
  - 사용자의 입력(코드 진행)을 받으면 [기능 분석 -> 기법 탐지 -> 연주 가이드] 순으로 도구를 체이닝하여 종합적인 답변을 구성합니다.

### 3. Streamlit UI 및 인터랙션
- **사고 과정 시각화**: `st.status`를 활용하여 에이전트의 분석 단계(Thought/Action/Observation)를 실시간으로 노출합니다.
- **지능형 후속 질문**: 분석 종료 후 맥락을 분석하여 다음 단계의 궁금증을 유발하는 질문 3가지를 버튼으로 제공합니다. (예: "트라이톤 대리로 바꿔보면?", "이 진행을 쓴 유명한 곡은?")
- **이미지 분석 (Vision)**: 악보 이미지가 업로드된 경우 Gemini의 Vision 능력을 활용하여 텍스트 코드로 변환 후 분석 프로세스로 연결합니다.

### 4. 품질 및 예외 처리
- **분석 실패 차단**: 기술적 오류로 분석이 불가능할 경우, 절대로 RAG를 오용하여 가짜 조언을 지어내지 말고 실패 사실만 보고합니다.
- **컨텍스트 관리**: 최근 대화 기록 요약 및 수준별 정수 매핑(1~5)을 통해 효율적인 토큰 관리와 정밀한 필터링을 수행합니다.
- **GEMINI.md 준수**: `[Design_Docs]/GEMINI.md`에 명시된 모든 개발 표준 및 Single Source of Truth 원칙을 준수합니다.

## 기술 스택
- **Framework**: LangChain, Streamlit
- **LLM**: Google Gemini 3.5 Flash (Multi-modal 지원)
- **VectorDB**: FAISS (with YAML Metadata Filtering)
- **Environment**: Python 3.9+ / Windows & Linux 지원
